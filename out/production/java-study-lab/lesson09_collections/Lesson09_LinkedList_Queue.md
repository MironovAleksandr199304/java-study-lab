# Lesson09 — LinkedList и Queue

## 1. Зачем они нужны
ArrayList хорош, когда нужно быстро извлекать по индексу, но плох, если приходится часто вставлять или удалять элементы в середине.  
LinkedList решает эту проблему — он устроен как цепочка узлов, где каждый элемент знает, кто следующий и кто предыдущий.

Если представить в голове:
[A] ↔ [B] ↔ [C] ↔ [D]
— то вставка нового элемента между B и C просто меняет ссылки.

## 2. Устройство LinkedList
Каждый элемент — это объект с тремя полями:
- value — данные,
- next — ссылка на следующий элемент,
- prev — ссылка на предыдущий.

Поэтому LinkedList медленнее ищет элемент (O(n)), но быстро вставляет и удаляет (O(1) при известной позиции).

## 3. Queue — очередь
Queue (интерфейс) реализует принцип FIFO: First In — First Out (“первым пришёл — первым ушёл”).  
Примеры из жизни:
- очередь сообщений в Kafka или RabbitMQ;
- очередь задач в CI/CD;
- очередь действий в UI-тестах (click, type, scroll).

Чаще всего используют:
- LinkedList — простая очередь,
- PriorityQueue — элементы отсортированы по приоритету,
- ArrayDeque — быстрая двусторонняя очередь.

## 4. Основные методы Queue
| Метод | Действие |
|-------|-----------|
| add() | добавляет элемент (ошибка, если нет места) |
| offer() | добавляет элемент, возвращает false, если не удалось |
| poll() | достаёт и удаляет элемент из головы |
| peek() | достаёт, но не удаляет |
| remove() | удаляет из головы, если пусто — ошибка |

## 5. Пример
```java
import java.util.LinkedList;
import java.util.Queue;

public class ExampleQueue {
    public static void main(String[] args) {
        Queue<String> tasks = new LinkedList<>();

        tasks.offer("Открыть браузер");
        tasks.offer("Авторизоваться");
        tasks.offer("Отправить форму");

        System.out.println("Первый шаг: " + tasks.peek()); 
        System.out.println("Выполняем: " + tasks.poll());  

        System.out.println("Осталось: " + tasks);
    }
}
```
Результат:
```
Первый шаг: Открыть браузер
Выполняем: Открыть браузер
Осталось: [Авторизоваться, Отправить форму]
```

## 6. Применение в QA
- Очередь шагов теста в автотесте.
- Очередь логов для последовательной обработки.
- Очередь запросов к API при нагрузочном тесте.
